# Archive layout (v4.7+)

The immutable nightly archive at `<config-root>/archive/` is the substrate for overnight learning. Written by `/listen`, read by mining agents and (optionally) re-mineable on demand. Never written by interactive commands — only by `/listen`.

## Why immutable

Karpathy LLM-wiki pattern: separate the raw source room (read-only, never re-written) from the extracted knowledge (LLM-owned, evolves over time). Three benefits:

1. **Reproducibility.** If mining produced a wrong proposal, you can re-mine yesterday's archive with different settings.
2. **Audit.** "What did I do on 2026-05-12?" answerable without rebuilding from external connectors that may rate-limit or change.
3. **Privacy boundary.** Archive lives on local disk under `<config-root>/`. Everything that came from external systems lands here once and is then mined locally.

## Layout

```
<config-root>/archive/
├── YYYY-MM-DD/
│   ├── calendar.md         # day's events (one section per event)
│   ├── inbox.md            # threads where user was in To/Cc, metadata only by default
│   ├── slack.md            # mentions + authored messages across watched channels
│   ├── drive.md            # files modified / created in watched folders
│   ├── transcripts/
│   │   ├── <meeting-id>-<slug>.md
│   │   └── ...
│   └── _index.md           # one-page summary: counts, sources pulled, errors if any
├── YYYY-MM-DD/
├── ...
└── _retention.md           # spec for archive retention/pruning policy
```

Each day's directory is **append-only after `/listen` writes it**. `/listen` refuses to overwrite an existing day's directory unless invoked with `--rewrite YYYY-MM-DD` (rare; for fixing broken pulls).

## Per-file structure

### `calendar.md`
```markdown
# Calendar — YYYY-MM-DD

## HH:MM — <title>
- **With:** <attendee list>
- **Location:** <video link or "—">
- **Duration:** <N> min
- **Source:** <calendar-id> · <event-id>
```

### `inbox.md`
Default privacy: **metadata only** (sender, subject, first 200 chars, message ID). Full bodies optional via `<config-root>/plugins/cortex.user-context.md` setting `listen.inbox.full_body: true`.

```markdown
# Inbox — YYYY-MM-DD

## <sender-name> — <subject>
- **Received:** <ISO timestamp>
- **Thread:** <thread-id>
- **Snippet:** <first 200 chars>
- (full body if config opt-in)
```

### `slack.md`
Only messages where the user was @mentioned, or messages the user authored, in channels the user has joined. Per-channel grouping.

### `drive.md`
Files created or modified yesterday in folders the user opted into via `/setup-sources`. Metadata only (file name, path, last modifier).

### `transcripts/<meeting-id>-<slug>.md`
One file per transcript. Front-matter with source adapter (Granola / Gemini / Fireflies / Otter / generic-Drive), original ID, meeting title, participants, date. Body is the transcript text. The conversation-miner reads this.

### `_index.md`
```markdown
# Archive index — YYYY-MM-DD

Generated by /listen at <timestamp>.

## Counts
- Calendar events: <N>
- Inbox threads: <N>
- Slack mentions: <N>
- Drive changes: <N>
- Transcripts: <N>

## Sources pulled
- Calendar MCP: ok
- Gmail MCP: ok
- Slack MCP: skipped (not configured)
- Granola adapter: ok (<N> meetings)
- ...

## Errors
- (none) or list of errors with retry hints
```

## Retention

Default policy (configurable via `<config-root>/plugins/cortex.user-context.md`):

- **Last 30 days:** keep in `archive/YYYY-MM-DD/` directly.
- **31-180 days:** compress per-day directory to `archive/YYYY-MM.tar.gz` (one monthly tarball).
- **>180 days:** archive retention is user-driven. `/listen --retention prune` walks old tarballs and offers per-month delete confirmation. No silent deletion.

Compression and pruning happen in `/listen`'s tail (Step 7 below).

## Re-mining

`/listen --remine YYYY-MM-DD` re-runs mining against an existing archived day without re-pulling sources. Useful when:
- A mining agent updated and you want to re-extract.
- You changed `<config-root>/memory/` schema and want new proposals against the same raw substrate.
- A previous run failed mid-mine.

Re-mining writes a new `.commit-drafts/<date>-remine-<seq>.md` file instead of overwriting the original draft.

## What does NOT live in the archive

- Memory nodes (`<config-root>/memory/`) — extracted knowledge, evolves over time, not immutable.
- Plugin user-context files — config, not source data.
- Cortex commit-triage log — meta, not raw.
- Cowork artifact state — ephemeral.

The archive is **only** raw substrate from external systems on a given day. If it didn't come from a connector, it doesn't belong here.

## Gitignore

If `<config-root>/` is synced to a git repo, **add `archive/` to `.gitignore`**. The archive contains email snippets, Slack message text, and full transcripts. It's the most privacy-sensitive directory in `<config-root>/`. Setup commands should print this warning the first time `/listen` produces an archive.
